================
Model evaluation
================

CloudnetPy has work-in-progress model evaluation functionality. It aims to
connect Cloudnet observation products with simulation products from various
models and compare them. The main goal of the system is to highlight the
capabilities and shortcomings of different simulated cloud variables.

Model evaluation analysis will be done with several different time scales. The shortest
and the most important products are day scale downsampled products.
The day scale products are not yet the model evaluation products, but day scale
is the key step to create evaluation analysis between observations and model simulations.
At a later state of software development, time scales of month, season, year and decade
will also be created based on the day scale products.

The time-height grid of observation products is always finer than the grid of the models, so
observation data is downsampled by averaging to match the model grid.
Downsampled observations and simulations are L3 day scale products.

To ensure the most suitable observation downsampling, there are several versions of each
specific product. For all products there is a downsampled version on the height-advection
time grid. For cloud fraction there are several methods for generating the observation and
also a high-level-cloud-filtered version. For IWC, there are different masked versions.
